Prerequisites

What You Need Before Starting

Gather your tools. This takes about 15–20 minutes and you only do it once.

01

Create a Figma Account

Go to figma.com and sign up for free. The free plan works for this guide, though some advanced features (Code Connect, unlimited Dev Mode) require a paid plan.

figma.com Free tier OK
02

Download the Figma Desktop App

Go to figma.com/downloads and install the desktop app. The MCP server connection requires the desktop app — the browser version won't work for this workflow.

⚠️

Do not skip this. The web version of Figma cannot run the local MCP server that Claude connects to.

03

Create a Claude Account

Go to claude.ai and sign up. A free account gives you access to Claude chat. You'll also need Claude Code for the later phases.

04

Install Node.js

Claude Code runs via npm, which comes with Node.js. Download it from nodejs.org — choose the LTS (Long Term Support) version.

After installing, verify it worked by opening your terminal and running:

node --version npm --version

You should see version numbers for both. If you do, you're ready.

05

Install Claude Code

In your terminal (Mac: Terminal app / Windows: PowerShell), run:

npm install -g @anthropic-ai/claude-code

Then log in with your Anthropic account:

claude login

This opens a browser window to authenticate. Complete it and return to your terminal.

💡

The -g flag installs Claude Code globally, so you can use it from any folder on your computer.

Phase 1

Define Your Idea with Claude

Before touching Figma, use Claude chat to sharpen your idea into a structured design brief. This is where Claude becomes your thinking partner.

01

Start a Claude Conversation

Go to claude.ai and open a new chat. You're going to use Claude to turn a rough idea into a complete design brief.

02

Describe Your App Idea (Messy is Fine)

Tell Claude what you want to build in plain language. Don't worry about being precise — Claude handles vague inputs well. For example:

Your prompt to Claude I want to build a mobile app that helps busy parents track their kids' homework and school assignments. Something simple, calming, not stressful to look at. Kind of like a to-do app but specifically for school stuff with multiple kids.
03

Ask Claude to Generate a Design Brief

Once you've had a back-and-forth to refine the idea, ask Claude to structure it:

Your prompt to Claude Based on what we've discussed, write a full product design brief. Include: app name, target user, core problem, key screens (list each one), visual style direction (colors, tone, typography feel), and a one-line Figma Make prompt I can use to generate the first screen.
💡

The "Figma Make prompt" at the end is key. Claude will craft a precise, structured prompt for Figma's AI — which you'll use in Phase 4.

04

Save Your Brief

Copy the output into a document (Notion, Google Docs, or even a text file). You'll refer back to the key screens list and the Figma Make prompt throughout the rest of the process.

Phase 2

Map Your Flows in FigJam

Before designing screens, map out how users move through your app. Claude can generate these diagrams directly in FigJam.

01

Connect Figma to Claude (claude.ai connector)

In Claude.ai, click the plug icon (MCP Apps / Connectors) and search for Figma. Connect your Figma account. This is different from the Claude Code MCP setup — this is for the claude.ai web interface.

claude.ai → Connectors One-time setup
02

Create a New FigJam File

In the Figma desktop app, click New file → FigJam. Name it something like "App User Flows".

03

Ask Claude to Generate a User Flow Diagram

Back in Claude chat (with Figma connected), paste your brief and ask:

Your prompt to Claude Using my design brief, generate a user flow diagram in FigJam that maps out the full journey from onboarding to the core task of adding a homework assignment. Show all screens and decision points.

Claude will create the diagram directly on your FigJam canvas as editable shapes and arrows.

04

Review and Edit the Flow

Open the FigJam file. You'll see Claude's generated flow. Edit any nodes, add sticky notes with questions, and use the canvas to explore alternatives side by side. This is the most important thinking step — don't rush it.

💡

If Claude missed a screen or branch, just tell it: "Add a flow for the password reset path" and it will update the diagram.

Phase 3

Set Up Claude Code + Figma MCP

This is the technical backbone of the whole workflow. You're connecting Claude Code (in your terminal) to your Figma files. Do this once.

01

Open Your Terminal

On Mac: press Cmd + Space, type Terminal, press Enter.
On Windows: press Win + R, type powershell, press Enter.

02

Connect the Remote Figma MCP Server

Run this single command (this is the recommended method for most users):

claude mcp add --transport http figma https://mcp.figma.com/mcp
03

Authenticate Claude Code with Figma

Now open Claude Code:

claude

Once inside, type:

/mcp

Select Figma from the list. A browser window will open — click Allow access. Return to your terminal. You're now connected.

💡

To verify the connection worked, type /mcp again — you should see figma listed with a green connected status.

04

Alternative: Enable Desktop MCP in Figma (Optional)

If you're on an enterprise plan or prefer local connection:

  • Open Figma desktop app
  • Click the Figma logo (top left) → Preferences
  • Select Enable Dev Mode MCP Server
  • Note the local address shown: http://127.0.0.1:3845/sse
  • In your terminal run: claude mcp add --transport http figma-desktop http://127.0.0.1:3845/mcp
⚠️

Free Figma users are limited to ~6 uses/month with the desktop MCP. Use the remote MCP (Step 02) unless you have a specific reason for the desktop version.

05

Create a Project Folder

Create a folder on your computer for your project's code. Then navigate to it in your terminal:

# Mac / Linux mkdir my-app-prototype cd my-app-prototype # Windows (PowerShell) New-Item -ItemType Directory -Name my-app-prototype cd my-app-prototype

Launch Claude Code from inside this folder:

claude
Phase 4

Generate Hi-Fi Designs in Figma Make

Now use Figma's AI to turn Claude's structured prompts into actual screen designs on the canvas.

01

Open Figma Make

In the Figma desktop app, create a new file. Look for Figma Make in the toolbar or from the main menu. This is Figma's prompt-to-design feature.

💡

If you don't see Figma Make, make sure your Figma app is updated to the latest version.

02

Paste the Figma Make Prompt Claude Generated

Remember that "one-line Figma Make prompt" you asked Claude to generate in Phase 1? Paste it into Figma Make. For example:

Example Figma Make Prompt (Claude-generated) Design a calming mobile app dashboard for parents tracking kids' homework. Soft sage green and warm cream palette, rounded cards per child, progress bars per subject, bottom nav with Home / Add Task / Calendar / Settings. Clean sans-serif typography, gentle shadows, iOS-style layout.
03

Generate and Review

Figma Make renders the design onto your canvas. Review what it produced. Don't expect perfection on the first try.

04

Iterate with Follow-Up Prompts

Refine the design by prompting again. Examples:

Refinement prompts Make the cards more spacious with more breathing room between elements.

Add a floating action button for adding a new task — coral/orange color.

Make the typography larger and more readable for users over 35.
05

Generate All Key Screens

Go through your key screens list from Phase 1 and generate each one. Common screens to cover:

  • Onboarding / splash screen
  • Home / dashboard
  • Add item / form screen
  • Detail / single item view
  • Settings or profile screen
  • Empty state (no data yet)
06

Organize Screens on the Canvas

Arrange all your screens in a logical order on the Figma canvas, left to right following the user flow. Name each frame clearly (e.g. "01 — Home Dashboard", "02 — Add Task"). This makes the next phase much smoother.

Phase 5

Convert Designs to Code with Claude Code

Now Claude Code reads your Figma designs and generates real, working frontend code from them.

01

Copy the Link to a Figma Frame

In Figma, right-click any screen frame → Copy/Paste as → Copy link to selection. This gives you a URL like:

https://www.figma.com/design/ABC123.../My-App?node-id=1234-5678
02

Give Claude Code the Link and a Build Instruction

In your terminal (inside Claude Code), paste the link with a prompt:

Claude Code prompt Build this screen as a React component with Tailwind CSS. Match the layout, colors, spacing, and typography exactly. Make it responsive for mobile-first. Here is the Figma frame: [paste your link]
03

Review the Generated Code

Claude Code will generate component files in your project folder. Open the file in VS Code or any text editor to review. You can also ask Claude Code to preview it:

# Ask Claude Code to spin up a local preview Can you add a simple index.html that renders this component so I can preview it in my browser?
04

Repeat for Each Screen

Go screen by screen. As you build more components, Claude Code will notice patterns from your earlier components and maintain design consistency automatically.

💡

After a few screens, ask: "Extract the shared colors, fonts, and spacing into a design tokens file so all components stay consistent."

05

Use the Figma Specs Plugin for Precise Handoff

Install the free Figma Specs plugin from the Figma Community. Select a frame → run Specs → it generates a structured YAML file that gives Claude Code precise dimensions, spacing, and component hierarchy — resulting in much more accurate code output.

Figma Community → "Specs" Free plugin
Phase 6

Push Code Back to Figma Canvas

The "Code to Canvas" feature lets you send your live, working UI back into Figma as editable layers — so designers and developers see the same thing.

01

Use the generate_figma_design Tool

When your code is working and you want to push it to Figma, ask Claude Code:

Claude Code prompt Take the rendered version of the HomeScreen component and send it to my Figma canvas as editable layers using the generate_figma_design tool.
⚠️

This feature (Code to Canvas / generate_figma_design) requires the Remote MCP — not the desktop version — and is currently Claude Code-only.

02

Review in Figma

Open your Figma file. The code-generated UI now appears as native Figma layers — fully editable shapes, text, and components. The design team can make tweaks directly here, and the result feeds back into the next code iteration.

03

Compare Side by Side

Place the original Figma Make design and the code-generated version side by side on the canvas. This is your design QA step. Note the differences and fix them either in code or in Figma.

Phase 7

Iterate and Polish

This is where good prototypes become great ones. Use the connected loop between Claude and Figma to refine until it feels right.

01

Add Interactions in Figma Prototype Mode

In Figma, switch to Prototype mode (top-right panel). Connect your frames with arrows: tap Home → goes to Add Task, etc. Set transitions (slide, fade). This creates a clickable prototype you can share with anyone via a link — no code required.

02

Ask Claude to Review Your Prototype

Share your Figma file link with Claude (with the connector active) and ask:

Claude prompt Review this prototype for UX issues. Look at the user flows, screen hierarchy, and visual consistency. Flag anything that might confuse a first-time user. Here is the file: [Figma link]
03

Use Claude Code to Add Real Interactions

For a live, interactive prototype (not just click-through), ask Claude Code to add real functionality:

Claude Code prompt Add working state management to the HomeScreen so tasks can actually be checked off and the progress bar updates in real time. Keep everything in React local state for now.
04

Deploy Your Live Prototype (Optional)

Ask Claude Code to deploy your prototype to a shareable URL:

Claude Code prompt Deploy this prototype to Vercel so I can share a live URL with stakeholders for feedback. Set it up as a static React app.

Claude Code will generate all the config files needed and walk you through the Vercel deployment steps.

Common Troubleshooting

  • MCP not connecting: Make sure Figma desktop app is open and updated, then re-run the claude mcp add command
  • Claude Code not found: Close and reopen your terminal after installing via npm
  • Figma Make not visible: Update Figma desktop to the latest version
  • Code doesn't match design: Use the Figma Specs plugin to generate precise YAML before prompting Claude Code
  • Free plan limits: Remote MCP is the most generous for free users; desktop MCP caps at ~6 uses/month
